1. Requisitos Previos

Antes de comenzar, asegurate de que la PC cumpla con estos requisitos:

RequisitoVersion MinimaDescarga
WindowsWindows 10 (64 bits)---
Node.jsv18 LTS o superiorhttps://nodejs.org
PostgreSQLv14 o superiorhttps://postgresql.org/download
NSSMv2.24https://nssm.cc/release/nssm-2.24.zip
Nota: Si la PC destino ya tiene Node.js y PostgreSQL instalados, solo necesitas copiar el proyecto y configurar el servicio.

2. Instalar Node.js

1
Descargar el instalador

Ve a https://nodejs.org y descarga la version LTS (recomendada).

2
Ejecutar el instalador

Abre el archivo .msi descargado y sigue el asistente:

  • Acepta el acuerdo de licencia
  • Deja la ruta de instalacion por defecto: C:\Program Files odejs\
  • Asegurate de que este marcada la opcion "Add to PATH"
  • Finaliza la instalacion
3
Verificar la instalacion

Abre CMD y ejecuta:

node --version

Debe mostrar algo como v20.15.0 o superior.

3. Instalar PostgreSQL

1
Descargar PostgreSQL

Ve a https://postgresql.org/download y descarga el instalador para Windows.

2
Ejecutar el instalador
  • Selecciona los componentes: PostgreSQL Server y pgAdmin
  • Establece una contrasena para el usuario postgres (guardala, la necesitaras)
  • Deja el puerto por defecto: 5432
  • Finaliza la instalacion
3
Crear la base de datos

Abre pgAdmin o psql y crea la base de datos:

CREATE DATABASE ipsfa;
Importante: Si estas migrando desde otra PC, restaura el backup de la base de datos en lugar de crearla desde cero.

4. Copiar el Proyecto

Copia la carpeta completa del proyecto a la nueva PC. Por ejemplo:

Origen (PC anterior)Destino (PC nueva)
C:\Users\jhonny\Desktop\creditosC:\creditos
Contenido minimo que debe tener la carpeta:
  • server.js (o app.js) --- archivo principal del servidor
  • package.json --- dependencias del proyecto
  • src/ --- codigo fuente
  • database/ --- scripts SQL (opcional)

5. Instalar Dependencias

Abre CMD y navega a la carpeta del proyecto:

cd /d "C:\creditos"
npm install

Esto instalara todas las librerias listadas en package.json (express, pg, cors, etc.).

Resultado esperado: Veras mensajes de instalacion y al final added X packages.

6. Configurar la Base de Datos

Verifica que la configuracion de conexion en tu server.js (o archivo de configuracion) apunte a la nueva base de datos:

// Ejemplo de configuracion tipica
const dbConfig = {
  host: 'localhost',
  port: 5432,
  database: 'ipsfa',
  user: 'postgres',
  password: 'TU_CONTRASENA'
};
Atencion: Actualiza la contrasena si es diferente en la nueva PC.

7. Probar Manualmente

Antes de crear el servicio, prueba que el servidor arranca correctamente:

cd /d "C:\creditos"
node server.js

Si ves algo como:

Servidor corriendo en puerto 3000
Conectado a PostgreSQL

¡Todo funciona! Presiona Ctrl + C para detenerlo y continua con la instalacion del servicio.

Si da error: No continues. Revisa la configuracion de la base de datos y las dependencias antes de crear el servicio.

8. Descargar NSSM

NSSM (Non-Sucking Service Manager) convierte cualquier programa en un servicio de Windows.

1
Descargar NSSM

Ve a: https://nssm.cc/release/nssm-2.24.zip

Descarga el archivo ZIP.

2
Extraer el archivo correcto

Abre el ZIP. Dentro encontraras dos carpetas:

  • win32/ --- para Windows de 32 bits
  • win64/ --- para Windows de 64 bits (la mayoria)

Para saber cual necesitas, ejecuta en CMD:

wmic os get osarchitecture

Si dice 64-bit, usa win64/nssm.exe.

3
Colocar NSSM en el proyecto

Copia nssm.exe a la carpeta de tu proyecto:

C:\creditos\tools\nssm\nssm.exe

Crea las carpetas tools y nssm si no existen.

9. Crear el Servicio con NSSM

⚠️ Importante: Debes ejecutar CMD como Administrador para que NSSM pueda crear el servicio.
1
Abrir CMD como Administrador

Presiona Windows, escribe cmd, haz clic derecho en "Simbolo del sistema" y selecciona "Ejecutar como administrador".

2
Navegar al proyecto
cd /d "C:\creditos"
3
Abrir NSSM
"tools\nssm\nssm.exe" install IPSFAServer

Se abrira una ventana grafica de NSSM.

4
Completar los campos
CampoValor a ingresar
PathC:\Program Files\nodejs\node.exe
Startup directoryC:\creditos
Argumentsserver.js
NO pongas nssm.exe en Path. Debe ser la ruta de node.exe.
5
Instalar el servicio

Haz clic en el boton "Install service".

Si todo sale bien, veras un mensaje de confirmacion.

10. Configurar Auto-inicio y Logs

Desde CMD como Administrador, ejecuta:

Auto-inicio al encender la PC

"tools\nssm\nssm.exe" set IPSFAServer Start SERVICE_AUTO_START

Crear carpeta de logs

mkdir logs

Configurar archivos de log

"tools\nssm\nssm.exe" set IPSFAServer AppStdout "C:\creditos\logs\output.log"
"tools\nssm\nssm.exe" set IPSFAServer AppStderr "C:\creditos\logs\error.log"
¿Que son estos logs?
  • output.log --- todo lo que imprime console.log en tu servidor
  • error.log --- errores y excepciones del servidor

11. Iniciar el Servicio

net start IPSFAServer

Si todo esta correcto, veras:

El servicio IPSFAServer se ha iniciado correctamente.
Si da error: Revisa los pasos anteriores. El error mas comun es poner la ruta de nssm.exe en lugar de node.exe en el campo Path de NSSM.

12. Verificar que Funciona

Verificar estado del servicio

sc query IPSFAServer

Debe decir STATE: 4 RUNNING

Ver los logs

type "C:\creditos\logs\output.log"

Probar en el navegador

Abre tu navegador y ve a:

http://localhost:3000

(o el puerto que uses en tu server.js)

Verificar en el Administrador de Servicios

Presiona Win + R, escribe services.msc y busca "IPSFAServer". Debe decir "En ejecucion".

14. Detener el Servicio

Para detener el servicio temporalmente (sin eliminarlo):

Metodo 1: Desde CMD (como Administrador)

net stop IPSFAServer

Metodo 2: Desde el Administrador de Servicios

  1. Presiona Win + R, escribe services.msc y Enter
  2. Busca "IPSFAServer" en la lista
  3. Haz clic derecho --- "Detener"

Metodo 3: Desde NSSM

"tools\nssm\nssm.exe" stop IPSFAServer
Nota: Detener el servicio no lo elimina. Al reiniciar la PC, se volvera a iniciar automaticamente si esta configurado en SERVICE_AUTO_START.

15. Eliminar el Servicio

Para eliminar completamente el servicio (por ejemplo, si quieres reinstalarlo):

Metodo 1: Con NSSM (recomendado)

"tools\nssm\nssm.exe" remove IPSFAServer confirm

Metodo 2: Con Windows (sc)

sc delete IPSFAServer
⚠️ Atencion: Eliminar el servicio no borra tu proyecto ni la base de datos. Solo elimina la configuracion del servicio de Windows.

16. Solucion de Problemas

Error 193: "No es una aplicacion Win32 valida"

Causa: El campo Path en NSSM apunta a nssm.exe en lugar de node.exe.

Solucion: Elimina el servicio y reinstalalo con la ruta correcta de Node.js:
C:\Program Files\nodejs\node.exe

Error 1060: "El servicio especificado no existe"

Causa: El servicio no se instalo correctamente o ya fue eliminado.

Solucion: Reinstala el servicio siguiendo los pasos 9 a 11.

"Acceso denegado" al ejecutar NSSM

Causa: No estas ejecutando CMD como Administrador.

Solucion: Cierra CMD, abrelo de nuevo como Administrador (clic derecho --- Ejecutar como administrador).

El servicio se instala pero no arranca

Posibles causas:
  • Error en el codigo de server.js
  • La base de datos no esta configurada correctamente
  • Falta alguna dependencia (npm install no se ejecuto)
Diagnostico: Revisa el archivo C:\creditos\logs\error.log para ver el error exacto.

NSSM no se ejecuta ("No se puede ejecutar esta aplicacion")

Causa: Descargaste la version incorrecta de NSSM (32 bits en Windows 64 bits, o viceversa).

Solucion: Verifica tu arquitectura con wmic os get osarchitecture y descarga la version correcta (win32 o win64).

El servicio no se inicia automaticamente al encender la PC

"tools\nssm\nssm.exe" set IPSFAServer Start SERVICE_AUTO_START
